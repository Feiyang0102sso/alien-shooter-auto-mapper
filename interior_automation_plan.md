# 自动化内饰阶段方案

## 背景

下一个大阶段要处理地图内饰自动化。内饰类型很多，粗略分就包括：

- 办公室组合
- 书架组合
- 大型计算机组合
- 实验室培养容器
- 实验室器材组合

这些组合还需要考虑朝向。但内饰和墙体不同，它没有强制的几何对齐关系。能对齐会更整齐，但对不齐也不会影响地图功能。

因此内饰系统不应该按墙体那种强规则、强吸附、强拼接的方式设计。

## 核心判断

墙体和门属于结构生成：

```text
强几何、强对齐、强规则
```

内饰属于视觉布置：

```text
弱几何、弱对齐、视觉自然优先
```

所以内饰阶段不应该追求“自动装修整张地图”，而应该优先做成：

```text
生成、保存、复制、粘贴、微调一批内饰组合块
```

也就是一个 `Interior Stamp` 系统。

## 推荐方向：Interior Stamp

新版编辑器会有批量复制功能，实际工作流通常是：

```text
复制一堆物件 -> 粘贴到空地图 -> 手动调整位置
```

这个思路非常适合内饰自动化。系统不必从零理解“如何装修”，而是把人工摆好的组合沉淀为可复用的 stamp。

推荐流程：

```text
手动摆出好看的内饰组合
框选这一组物件
保存为 stamp
以后从 stamp 库里批量粘贴
粘贴时允许旋转、随机偏移、随机删减或替换小件
```

这样能同时解决两个问题：

- 组合足够多：靠手工样板逐步积累。
- 不容易重复：靠旋转、偏移、删减、替换制造变体。

## 为什么不直接手写大量组合

如果直接写几十上百套完整组合，会很快失控：

```text
office_set_01
office_set_02
office_set_03
...
```

更合理的方式是：

```text
少量骨架组合 + 大量变体层
```

例如办公室区域不需要手写 50 套完整组合，而是拆成：

- 办公室骨架 5 套
- 桌面物件变体 10 套
- 椅子位置变体 4 套
- 杂物散布变体 8 套
- 朝向 4 种
- 轻微随机偏移

组合起来后，视觉变化会远大于实际维护成本。

## 内饰组合的分层

一个内饰 stamp 可以拆成几层理解：

```text
骨架层：决定大件，例如桌子、书架、服务器柜、培养罐、操作台
附属层：决定功能感，例如电脑、键盘、屏幕、仪器、箱子、文件
杂物层：制造自然差异，例如纸张、椅子偏移、血迹、破碎物、桶
缺失层：允许部分小件不出现
朝向层：同一组合旋转后形成不同视觉
```

原则：

```text
大件决定秩序
小件制造自然
```

例如实验室培养罐组合：

- 骨架可以是单培养罐、双培养罐、三培养罐横排、培养罐加操作台。
- 变体可以来自旁边仪器、灯、箱子、血迹、管线、缺失设备。

## 对齐策略

内饰不需要全局强对齐，只需要分级处理：

```text
贴墙类：尽量沿墙排齐
中心类：只要求整体在房间里
杂物类：允许轻微随机散开
```

示例：

- 书架组合：贴墙，朝向墙内，位置可以轻微吸附到 20 或 40 像素步长。
- 办公桌组合：可以贴墙，也可以靠中间，椅子、纸张、电脑允许小偏移。
- 实验室培养罐：大物件本体稍微对齐，周围仪器和管线可以不对齐。
- 尸体、纸张、箱子、杂物：不应该过度对齐，随机一点反而自然。

## 朝向策略

位置可以弱对齐，但朝向必须统一抽象。否则组合多了以后会失控。

建议先支持 4 个逻辑朝向：

```text
NORTH
SOUTH
WEST
EAST
```

每个 stamp 默认按一个方向保存。粘贴时统一做旋转变换：

```text
本地坐标 -> 根据朝向旋转 -> 加到目标锚点 -> 生成 sprite
```

不要为每个方向手写一套坐标。

## 数据结构草案

建议的核心结构：

```text
InteriorStamp
- id
- category
- anchor_type
- width
- height
- props

StampProp
- vid
- local_x
- local_y
- local_z
- direction
- role
- required
- jitter_x
- jitter_y
- replace_group
```

字段说明：

- `required`: 是否为核心物件。桌子、书架、培养罐等大件一般为 `true`。
- `role`: 物件角色，例如 `core`、`surface`、`clutter`、`wall`、`floor`。
- `replace_group`: 替换组，例如 `monitor`、`chair`、`box`、`lab_tool`。
- `jitter_x / jitter_y`: 粘贴时允许的小范围随机偏移。

## 系统模块建议

命名上可以避免使用过于泛化的 `InteriorBuilder`，更准确的拆法是：

```text
InteriorStampLibrary
InteriorStampPlacer
InteriorStampMutator
```

职责划分：

- `InteriorStampLibrary`: 负责保存、读取、分类 stamp。
- `InteriorStampPlacer`: 负责平移、旋转、朝向转换、输出 sprite。
- `InteriorStampMutator`: 负责随机删减、替换小件、添加轻微偏移。

最终生成流水线：

```text
WallBuilder -> 墙、地板、天花板 sprites
DoorBuilder -> 门 sprites
InteriorStampPlacer -> 内饰 sprites

全部合并 -> MapWriter
```

内饰应该是独立 sprite 生成层，不要污染已经稳定的墙体和门逻辑。

## 手动微调边界

新版编辑器里的批量复制/粘贴可以作为内饰 stamp 的基础工作流，但需要明确一个限制：

```text
粘贴后的手动微调，只适合调整整块 stamp 的上下左右平移。
```

也就是说，人工后处理通常只能做：

- 整体向上、向下、向左、向右移动。
- 把整块组合放到更合适的位置。
- 少量兜底修正。

人工后处理不适合做：

- 调整每个物件之间的间距。
- 单独移动某个小物件。
- 改变整块组合的方向。
- 粘贴后再整体旋转。

原因是组合内部的相对位置和方向一旦确定，后期整体旋转或强行改方向，很容易让桌椅、书架、仪器、培养罐之间的视觉关系变乱。

因此 `InteriorStampMutator` 的职责应该前移：

```text
变体必须在粘贴前生成
粘贴后主要只允许整体平移
自动化阶段尽量一次生成可用结果
```

如果确实需要改变内部间距或方向，应优先通过以下方式解决：

- 保存多个不同方向的 stamp 样板。
- 保存多个不同间距的 stamp 变体。
- 在生成阶段对子物件应用 jitter。
- 在生成阶段对子物件做可控替换或删减。
- 以后如果子动画系统能支持，再考虑用子动画处理局部变化。

这意味着 stamp 库不能只保存“一份组合然后到处旋转”。更稳的做法是：

```text
重要组合手工保存多个朝向版本
系统负责选择合适版本
系统只做小范围随机变化
人工最后只做整块平移
```

换句话说，stamp 系统不能把主要调整压力留给编辑器。编辑器里的手动调整比较麻烦，所以自动化阶段需要提前完成大部分决策：

- 选哪一个 stamp。
- 用哪个朝向版本。
- 放在哪个锚点。
- 内部小件是否删减、替换、偏移。
- 是否避开门口、墙体和已有物件。
- 是否给大型物件留出基本视觉空间。

人工微调只能作为最后保险，而不是主要流程。

因此每次生成 stamp 前，系统应该尽量做一次简单评分：

```text
位置合适度
朝向合适度
是否挡门
是否穿墙
是否和已有内饰重叠
视觉重复度
```

评分不需要复杂，但必须能过滤明显糟糕的结果。宁愿少放一个组合，也不要生成一个一眼就别扭、还需要大量手动修的组合。

## 分阶段任务

### V1：批量复制和保存 stamp

目标是先服务手动工作流。

任务：

- 支持框选一组物件。
- 支持将选中物件保存为 stamp。
- stamp 内记录相对坐标、VID、Z、方向。
- 支持从 stamp 库选择并粘贴到地图。

### V2：stamp 旋转和随机偏移

目标是让同一个 stamp 可以适配不同墙边和房间位置。

任务：

- 支持 4 个方向旋转。
- 粘贴时允许整体轻微偏移。
- 单个非核心物件允许 `jitter`。
- 先做简单的出界和穿墙检查。

### V3：required / optional / replace_group

目标是降低重复感。

任务：

- 核心物件必须保留。
- 杂物、小件、血迹、纸张可以按概率删除。
- 同类小件可以随机替换。
- 每次粘贴同一个 stamp，都能产生轻微差异。

### V4：半自动填充房间

目标是把手动粘贴升级为半自动布置。

任务：

- 根据房间大小选择候选 stamp。
- 根据墙边、角落、中心区域选择锚点。
- 避开门口和已有物件。
- 生成后仍允许人工微调。

## 第一批建议支持的 stamp 类型

不要一开始覆盖所有组合。先选能验证主要机制的类型：

- 办公室桌面组合
- 书架贴墙组合
- 实验室培养罐组合

这三类分别覆盖：

- 中心或贴墙摆放
- 强贴墙和朝向需求
- 大物件占用和周围杂物变体

跑通后再扩展：

- 大型计算机组合
- 实验室操作台组合
- 仓储箱子组合
- 尸体和战斗痕迹组合
- 纯杂物散布组合

## 最终原则

内饰自动化的目标不是让程序完全替代人工审美，而是：

```text
复用已经摆好的好看组合
批量粘贴到新地图
自动制造足够多的不重复变体
最后保留人工微调空间
```

换句话说，现阶段最应该做的不是“全自动装修器”，而是：

```text
可积累、可变体、可批量粘贴的内饰印章库
```
